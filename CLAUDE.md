# CLAUDE.md — контекст проекта для Claude Code

Этот файл описывает проект, принятые решения и рабочие сценарии. Читается
Claude Code автоматически.

## Что это

Статический сайт-карта пеших маршрутов по Грузии на Leaflet. Хостится на GitHub
Pages, без сервера и без API-ключей. Вдохновлён nakarte/nageokarte, но это
самостоятельная лёгкая реализация (не форк).

Возможности: карта с переключением безключевых слоёв, поиск по названию, фильтр
по длине, режим «только в видимой области карты», автораскраска треков по
авторам, кликабельная легенда авторов (скрытие/показ), попап с деталями и
ссылкой на источник.

## Структура

```
geo-tracks/
├─ tracks/                 # исходные .gpx (сейчас 1253 шт.) — источник правды
├─ build/
│  ├─ build.py             # сборка tracks/ → site/data/*.json
│  ├─ overrides.csv        # ручные правки названий (file,location)
│  └─ requirements.txt     # только для --elevation (requests)
├─ site/                   # КОРЕНЬ САЙТА (то, что публикуется)
│  ├─ index.html
│  ├─ css/style.css
│  ├─ js/app.js            # vanilla JS, без сборщика/зависимостей
│  └─ data/                # ГЕНЕРИРУЕТСЯ build.py — не править руками
│     ├─ index.json        # лёгкий индекс: имя, длина, автор, цвет, bbox, center
│     ├─ geometry.json     # {id: [[lat,lon],...]} упрощённая геометрия
│     └─ meta.json         # count, authors[], extent, view, sliderMax, maxLengthKm
└─ .github/workflows/deploy.yml   # автосборка + деплой на Pages
```

Leaflet подключается с CDN (unpkg) прямо в `index.html`. Сборщиков (npm, vite и
т.п.) НЕТ и не требуется. Пути в сайте относительные — работает в подпапке
`/geo-tracks/`.

## Поток данных

`tracks/*.gpx` → `build/build.py` → `site/data/*.json` → `site/js/app.js` рисует.

`build.py` для каждого GPX:
- парсит точки трека и ссылку-источник из `<metadata>`;
- отрезает авторский суффикс из имени файла (словарь `AUTHORS`, замкнутый набор
  из 26 авторов);
- нормализует имя в формат `location_name__length__index` (убирает номера
  маршрута, метрики в скобках, приводит к Title Case);
- считает длину по гаверсинусу из координат;
- упрощает геометрию (Douglas–Peucker, `--eps`, по умолчанию ~7 м);
- назначает цвет по автору (хеш → палитра);
- опционально (`--elevation`) тянет набор высоты с Open-Meteo.

## Рабочие команды

Локальный просмотр (по file:// данные не грузятся — нужен http):
```bash
cd site && python3 -m http.server 8000   # → http://localhost:8000
```

Пересборка данных после изменений в tracks/:
```bash
python3 build/build.py
```

С набором высоты (медленно, сотни запросов, значения приблизительные):
```bash
pip install -r build/requirements.txt
python3 build/build.py --elevation
```

## Как добавить новые маршруты

1. Положить `.gpx` в `tracks/`.
2. `python3 build/build.py`.
3. Закоммитить и запушить — Actions пересоберёт и опубликует.

Если автор нового файла не из списка — суффикс останется в названии; добавить
автора в словарь `AUTHORS` в `build/build.py`. Кривые имена правятся через
`build/overrides.csv` (строка `имя_файла.gpx,Желаемое название`).

## Деплой

Публичный репозиторий + Settings → Pages → Source: **GitHub Actions**. Workflow
`.github/workflows/deploy.yml` запускает `build.py` и публикует папку `site/` на
`https://<username>.github.io/geo-tracks/`.

## Ключевые решения и ограничения (важно помнить)

- **Только безключевые open-source слои**: OpenTopoMap, OpenStreetMap, CyclOSM,
  OSM Humanitarian, Esri World Imagery (спутник), Esri World Topo + оверлей троп
  Waymarked. Яндекс/Google и прочее с ключами НЕ используем — осознанно.
- **Чего нет в исходных GPX**: высот (`<ele>`), реальных времён, точек интереса,
  сложности, типа активности, длительности. Поэтому таких фильтров нет и быть не
  может без внешних источников. Доступные фильтры: название, длина, видимая
  область, автор (через легенду). Высота — только через `--elevation`.
- **Длина**: считается из координат. По просьбе пользователя **удалены все
  маршруты длиннее 50 км** (было 1286 → стало 1253, максимум 49.7 км). Если
  снова появятся длинные треки — это сознательно нежелательно.
- **Слайдер длины**: верхняя граница = 95-й перцентиль (`sliderMax` в meta.json),
  на максимуме работает как «и длиннее».
- **Стартовый вид карты**: по «ядру» данных (перцентили центров треков, поле
  `view` в meta.json), а не по полному extent — чтобы редкие выбросы (пара
  треков в Турции/на Памире) не отзумливали карту на пол-Азии.
- **Имена**: транслит с грузинского, местами длинные/задвоенные — это из самих
  файлов. Массово не «причёсываются», точечно — через overrides.csv.
- **Производительность**: 1253 полилинии рисуются через `L.canvas`
  (preferCanvas). Список в сайдбаре пагинируется по 60. Поиск с дебаунсом 160 мс.

## Стек и стиль кода

- Фронт: чистый HTML/CSS/JS, без фреймворков и сборки. Leaflet 1.9.4 с CDN.
- Сборка: Python 3 stdlib (без зависимостей; `requests` только для --elevation).
- UI и комментарии — на русском.
- Эстетика: тёплая «топографическая/издательская» палитра, шрифты Fraunces +
  IBM Plex Sans с Google Fonts.
