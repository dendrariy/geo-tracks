# Тропы Грузии — карта маршрутов

Статический сайт-карта на Leaflet с предзагруженными GPX-маршрутами: поиск по
названию, фильтр по длине, режим «только в видимой области», автоматическая
раскраска треков по авторам. Хостится на GitHub Pages, без сервера и без
API-ключей.

```
geo-tracks/
├─ tracks/                 # исходные .gpx (1286 шт.) — сюда кладёте новые
├─ build/
│  ├─ build.py             # сборка данных из tracks/ → site/data/
│  ├─ overrides.csv        # ручные исправления названий (необязательно)
│  └─ requirements.txt     # нужен только для --elevation
├─ site/                   # то, что публикуется (корень сайта)
│  ├─ index.html
│  ├─ css/style.css
│  ├─ js/app.js
│  └─ data/                # генерируется build.py
└─ .github/workflows/deploy.yml
```

## Локальный просмотр

Из-за политики браузера данные нельзя открыть по `file://` — нужен локальный
сервер:

```bash
cd site
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Как добавить новые маршруты

1. Положите новые `.gpx` в папку `tracks/`.
2. Пересоберите данные:
   ```bash
   python3 build/build.py
   ```
3. Закоммитьте и запушьте — GitHub Actions сам пересоберёт и опубликует.

Имена файлов нормализуются автоматически в формат
`location_name__length__index`; авторский суффикс из имени отрезается. Если
автор у нового файла не из известного списка, он останется в названии — добавьте
его в словарь `AUTHORS` в `build/build.py`.

### Ручная правка названий

Если автонормализация дала кривое имя, впишите строку в `build/overrides.csv`:

```csv
file,location
adishi-iprari_Zviad_Kelenjeridze.gpx,Адиши — Ипрари
```

`file` — имя файла как в `tracks/`, `location` — желаемое название.

### Набор высоты (необязательно, медленно)

Высоты в исходных GPX отсутствуют. Их можно дотянуть с бесплатного сервиса
Open-Meteo (без ключа), значения приблизительные (SRTM, выборка точек):

```bash
pip install -r build/requirements.txt
python3 build/build.py --elevation
```

После этого в карточке трека появится «Набор высоты». Запуск делает сотни
запросов и занимает несколько минут.

## Публикация на GitHub Pages

1. Создайте **публичный** репозиторий с именем `geo-tracks`.
2. Запушьте в него содержимое этой папки (ветка `main`).
3. В репозитории: **Settings → Pages → Build and deployment → Source: GitHub
   Actions**.
4. После первого пуша workflow соберёт данные и опубликует сайт на
   `https://<username>.github.io/geo-tracks/`.

Сайт использует относительные пути, поэтому корректно работает в подпапке
`/geo-tracks/`.

## Слои карты (все без ключей)

OpenTopoMap, OpenStreetMap, CyclOSM, OSM Humanitarian, Esri World Imagery
(спутник), Esri World Topo; оверлей пеших троп Waymarked Trails. Переключатель —
в правом верхнем углу.

## Что можно и нельзя фильтровать

В данных есть: название, длина (считается из координат), автор, платформа-
источник, ссылка на оригинал. Фильтры: поиск по названию, диапазон длины,
видимая область карты, скрытие авторов (через легенду). Сложности, типа
активности и длительности в исходных файлах нет — этих фильтров быть не может.
Набор высоты — опционально (см. выше).
